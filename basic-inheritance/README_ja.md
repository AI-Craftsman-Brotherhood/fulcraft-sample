# basic-inheritance

## 概要

`PricingPolicy` インターフェースとその実装クラス `MemberPricingPolicy` の 2 ファイル構成で、インターフェース継承・実装関係の解析を FUL に行わせるサンプルである。型階層の依存解決と、実装クラスが持つコンストラクタバリデーション・計算ロジックを LLM がどのように文書化するかを検証できる。

## ソース構成

| 項目 | 値 |
|------|-----|
| ファイル数 | 2 |
| クラス / インターフェース数 | 2 |
| 公開メソッド数 | 2 |
| 総行数（空行含む） | 30 |

## 複雑度の特徴

- `PricingPolicy` はメソッド 1 件のみを持つ最小インターフェースで、複雑度の基準点として機能する
- `MemberPricingPolicy` のコンストラクタに範囲チェック（0〜50）の条件分岐、`applyDiscount` に price ガード + 整数割り算の分岐が存在する
- インターフェース → 実装クラスの 1 段継承で、FUL の動的解決（dynamic resolution）が型階層をどう追跡するかの確認対象
- フィールド `discountPercent` は `final` で不変

## FUL 機能との対応

| FUL 機能 | このサンプルでの検証ポイント |
|----------|--------------------------|
| ANALYZE  | インターフェースと実装クラス間の依存関係グラフ生成、`implements` 関係の動的解決の正確性確認 |
| DOCUMENT | インターフェース側の抽象メソッドと実装側のオーバーライドメソッドで LLM が生成する JavaDoc の内容差異の観察 |
| REPORT   | 2 ファイル構成での品質ゲート評価において、インターフェース単体とクラス単体がそれぞれ適切に採点されるかの確認 |
