# basic-inheritance

## Overview

A two-file sample consisting of the `PricingPolicy` interface and its implementation class `MemberPricingPolicy`, designed to have FUL analyze an interface inheritance and implementation relationship. It allows verification of how FUL resolves type-hierarchy dependencies and how the LLM documents the constructor validation and calculation logic of the implementation class.

## Complexity Characteristics

- `PricingPolicy` is a minimal interface with only one method, functioning as a baseline reference point for complexity
- `MemberPricingPolicy`'s constructor has a range-check conditional branch (0–50), and `applyDiscount` has a price guard plus an integer-division branch
- The single-level interface → implementation class inheritance is a subject for examining how FUL's dynamic resolution tracks the type hierarchy
- The field `discountPercent` is `final` and immutable

## FUL Feature Coverage

| FUL Feature | Verification Points in This Sample |
|-------------|-----------------------------------|
| ANALYZE  | Dependency graph generation between the interface and implementation class; accuracy of dynamic resolution for the `implements` relationship |
| DOCUMENT | Observing differences in JavaDoc content generated by the LLM for the abstract method on the interface side vs. the overriding method on the implementation side |
| REPORT   | Verifying that both the interface alone and the class alone are scored appropriately in a quality gate evaluation with a 2-file configuration |
