# basic-if-switch

## 概要

連鎖 `if` 文による範囲判定とアロー構文の `switch` 式による多値マッピングを同一クラスに持つ `DecisionService` を解析対象とするサンプルである。2 種類の分岐構造が共存するコードに対して、FUL が複雑度を正確に計上し、LLM が各分岐の意味をドキュメント化できるかを検証できる。

## ソース構成

| 項目 | 値 |
|------|-----|
| ファイル数 | 1 |
| クラス / インターフェース数 | 1 |
| 公開メソッド数 | 2 |
| 総行数（空行含む） | 29 |

## 複雑度の特徴

- `gradeLabel` は境界値チェック + 3 段階の `if` チェーンを持ち、条件分岐ごとに早期 return する
- `shippingZone` は null / blank ガードの後、複数ラベルを束ねたアロー構文 `switch` 式（Java 14+ 構文）を使用する
- `switch` の `case` 句で複数値を列挙するパターン（`"Tokyo", "Kanagawa", ...`）を Spoon / JavaParser がどう解析するかの確認対象
- ネスト深度は最大 1 段、外部依存なし

## FUL 機能との対応

| FUL 機能 | このサンプルでの検証ポイント |
|----------|--------------------------|
| ANALYZE  | 連鎖 `if` と `switch` 式それぞれに対するサイクロマティック複雑度の計上方式の比較、`--engine` 差異の観察 |
| DOCUMENT | 各分岐の戻り値ラベル（`"KANTO"` / `"KANSAI"` 等）を LLM が列挙形式で JavaDoc に記述できるかの確認 |
| REPORT   | 2 メソッドの複雑度が別々のカテゴリに分類されるケースを通じた品質ゲート評価の粒度確認 |
