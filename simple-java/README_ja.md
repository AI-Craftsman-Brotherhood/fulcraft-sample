# simple-java

## 概要

四則演算のみを持つ最小構成の単一クラスで、FUL が正常系のベースライン解析を行う際の基準サンプルとして機能する。複雑度・依存関係ともに最小限であり、ANALYZE・DOCUMENT・REPORT の各機能が期待通りの出力を返すかを最もシンプルな条件で検証できる。

## ソース構成

| 項目 | 値 |
|------|-----|
| ファイル数 | 1 |
| クラス / インターフェース数 | 1 |
| 公開メソッド数 | 4 |
| 総行数（空行含む） | 27 |

## 複雑度の特徴

- 分岐は `divide` メソッドの単一 `if` のみ（サイクロマティック複雑度は全体で最小クラス）
- ネスト深度は最大 1 段
- 例外送出は 1 箇所（`ArithmeticException`）、catch / finally なし
- クラス間依存ゼロ、継承なし、インターフェース実装なし

## FUL 機能との対応

| FUL 機能 | このサンプルでの検証ポイント |
|----------|--------------------------|
| ANALYZE  | 複雑度スコア最小値の算出精度、`--engine` オプション（composite / javaparser / spoon）間の出力一致確認 |
| DOCUMENT | JavaDoc が存在しない状態から LLM が 4 メソッド分のドキュメントを生成する際の品質と網羅性の確認 |
| REPORT   | 品質ゲートの合格ベースラインとして機能し、`--max-cyclomatic` 閾値設定の動作確認に利用できる |
