# basic-collections

## 概要

`List` と `Map` を入力として受け取り、カテゴリ集計・低在庫判定・合計算出を行う `InventoryService` を解析対象とするサンプルである。コレクション操作と null ガード分岐が複合する実装パターンに対して、FUL の静的解析とドキュメント生成がどのように機能するかを検証できる。

## ソース構成

| 項目 | 値 |
|------|-----|
| ファイル数 | 1 |
| クラス / インターフェース数 | 1 |
| 公開メソッド数 | 3 |
| 総行数（空行含む） | 49 |

## 複雑度の特徴

- メソッドごとに null ガード + ループ + 条件分岐の 3 層構造を持ち、サイクロマティック複雑度がベースラインより明確に高い
- `for-each` ループ内に `continue` / 早期 `return` の脱出パターンが混在する
- `Map.Entry` の反復処理と `getOrDefault` による複合的なコレクション操作
- クラス間依存なし、継承なし、標準ライブラリ（`java.util`）のみ使用

## FUL 機能との対応

| FUL 機能 | このサンプルでの検証ポイント |
|----------|--------------------------|
| ANALYZE  | ループ + 分岐の複合パターンに対する複雑度スコア算出、Spoon と JavaParser の依存解決結果の比較 |
| DOCUMENT | null 許容パラメータや早期 return を持つメソッドに対して LLM が前提条件・戻り値を正確に記述できるかの確認 |
| REPORT   | `--complexity-strategy warn` 適用時に中程度の複雑度として分類されるかの閾値動作検証 |
