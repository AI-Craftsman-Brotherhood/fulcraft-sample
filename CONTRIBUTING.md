# Contributing Guide

This repository is a companion sample set for FUL and follows the same governance and contact channels.

## What You Can Do
- Report reproducible bugs in the sample code.
- Suggest improvements for sample readability or coverage through issue reports.
- Ask questions about sample behavior and intended usage.

## What You Cannot Do
- Submit code changes via pull request.
- Redistribute modified copies of this repository.
- Use this repository for commercial purposes without explicit written permission.

## Reporting Issues
When reporting an issue, include:
- The case directory name (for example, `oop-strategy-pattern`).
- Steps to reproduce.
- Expected behavior and actual behavior.
- Java version and execution environment.

## Security Issues
Do not post sensitive vulnerabilities in public issues.
Use the reporting channels in [SECURITY.md](SECURITY.md).
