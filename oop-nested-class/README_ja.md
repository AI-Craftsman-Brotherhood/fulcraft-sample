# oop-nested-class

## 概要

`Schedule` クラス内に静的ネストクラス `Slot` を持つ単一ファイル構成のサンプルです。FUL が1ファイルから複数の型定義（外部クラス＋ネストクラス）を検出・分析できるかを検証するターゲットとして機能します。

## ソース構成

| 項目 | 値 |
|------|-----|
| ファイル数 | 1 |
| クラス / インターフェース数 | 2（`Schedule`、`Schedule.Slot`） |
| 公開メソッド数 | 5（`addSlot`、`totalMinutes`、`titles`、`Slot.title`、`Slot.minutes`） |
| 総行数（空行含む） | 49 |

## 複雑度の特徴

- 外部クラスと静的ネストクラスが1ファイルに共存する2層構造
- ネストクラス `Slot` はコンストラクタで `null` チェックと負値排除を行う防御的正規化
- `totalMinutes()` / `titles()` で `for-each` ループによるリスト走査
- `final` フィールドのみで構成されるイミュータブルな値オブジェクト（`Slot`）
- クラス間依存: `Schedule` が `Slot` を生成・集約する一方向依存

## FUL 機能との対応

| FUL 機能 | このサンプルでの検証ポイント |
|----------|--------------------------|
| ANALYZE  | ネストクラスを独立した型として検出できるか。JavaParser / Spoon それぞれがネスト構造をどう表現するかの差異確認 |
| DOCUMENT | `Schedule` と `Schedule.Slot` に対して個別に JavaDoc が生成されるか。ネスト関係がドキュメントに反映されるか |
| REPORT   | 単一ファイル・低複雑度サンプルとしてのベースライン品質スコア確認 |
